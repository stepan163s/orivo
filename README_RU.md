# Orivo

**Orivo** — нативный macOS-менеджер медиасервисов. Автоматически устанавливает, запускает и настраивает **TorrServer** и **Jackett**, открывает встроенную медиатеку на базе Lampa и обходит защиту Cloudflare без Docker и сторонних зависимостей.

---

## Требования

| | |
|---|---|
| **macOS** | 13.0 Ventura и новее |
| **Архитектура** | Apple Silicon (M1/M2/M3) и Intel |
| **Место на диске** | ~200 МБ |

---

## Установка

1. Скачайте `Orivo.app` из раздела [Releases](https://github.com/your-org/orivo/releases)
2. Перенесите в папку `/Applications`
3. При первом запуске macOS может показать предупреждение «Приложение загружено из интернета» — нажмите **Открыть**

---

## Первый запуск

### 1. Разрешение локальной сети

> [!IMPORTANT]
> При первом запуске macOS покажет системный диалог:
> **«Orivo хочет найти устройства в локальной сети»**
>
> Необходимо нажать **Разрешить** — это нужно для работы встроенных серверов приложения. Без этого разрешения сервисы не смогут принимать подключения.

Если вы случайно нажали «Не разрешать», исправить можно вручную:
**Системные настройки → Конфиденциальность и безопасность → Локальная сеть → включите Orivo**

### 2. Установка сервисов

1. На главном экране онбординга нажмите **Продолжить**
2. Orivo автоматически скачает последние версии TorrServer и Jackett, настроит и запустит их.

---

## Lampa

Нажмите кнопку **Открыть Lampa** — откроется встроенная Lampa.

Orivo автоматически настраивает всё необходимое: адрес TorrServer, адрес и API-ключ Jackett. Никаких ручных действий не требуется.

---

## Использование на других устройствах в локальной сети

Вы можете использовать запущенные на Mac серверы TorrServer и Jackett на телевизоре, планшете или другом компьютере в вашей домашней сети.

> [!TIP]
> Локальный IP-адрес вашего Mac можно быстро узнать и скопировать в настройках Orivo (**Настройки...**) в разделе **«Локальная сеть»**.

> [!NOTE]
> Названия служб (TorrServer, Jackett) на главном экране и в настройках кликабельны (обозначены иконкой `↗`) — нажатие на них мгновенно открывает их веб-интерфейсы в браузере.

Для этого в настройках приложения Lampa (или на сайте lampa.mx) на другом устройстве укажите:

1. В разделе **TorrServer**:
   - **Адрес TorrServer**: `http://<IP-адрес-вашего-Mac>:8090`
2. В разделе **Парсер**:
   - **Использовать парсер**: `Да`
   - **Тип парсера**: `Jackett`
   - **Адрес парсера**: `http://<IP-адрес-вашего-Mac>:8098/jackett`
   - **Ключ Jackett** (API-ключ/токен): `<ваш-API-ключ>` *(его можно скопировать из веб-интерфейса Jackett)*

> [!NOTE]
> Запросы парсера проксируются через порт `8098` вашего Mac, чтобы обойти ограничения CORS на внешних устройствах.

## Настройки работы приложения и Язык

В Orivo встроено меню настроек (**Настройки...**), которое позволяет настроить поведение программы:

- **Закрытие окна (Крестик)**: 
  - По умолчанию при закрытии основного окна Orivo сворачивается и продолжает работать в фоне (показывается иконка в строке меню / Menu Bar).
  - Если включить опцию **«Закрывать Orivo при закрытии окна»**, то при нажатии на красный крестик программа будет закрываться полностью, автоматически туша все фоновые службы (TorrServer, Jackett и др.).
- **Язык интерфейса**: 
  - В настройках доступен переключатель языка (**Русский / English**), который мгновенно меняет локализацию всего приложения.

## Настройка прокси в Jackett

В некоторых регионах может потребоваться прокси для корректной работы трекеров. Если раздачи не находятся — попробуйте настроить прокси:

1. Откройте панель управления Jackett: [http://127.0.0.1:9117](http://127.0.0.1:9117) *(или кликните по названию Jackett в Orivo)*
2. Прокрутите вниз до раздела **Jackett Configuration**
3. Задайте параметры прокси:
   - **Proxy type**: `HTTP`
   - **Proxy URL**: адрес вашего прокси-сервера (например, `127.0.0.1`)
   - **Proxy Port**: порт вашего прокси-сервера
4. Нажмите **Apply server settings**

## Архитектура сервисов

| Сервис | Порт | Назначение |
|---|---|---|
| **TorrServer** | `8090` | Торрент-стример. Открывает торренты без полного скачивания |
| **Jackett** | `9117` | Агрегатор торрент-трекеров |
| **CORS-прокси** | `8098` | Встроенный прокси для корректной работы Lampa |
| **FlareSolverr** | `8191` | Встроенный обходчик Cloudflare. Не требует Docker |

## Логи и диагностика

Все логи сохраняются автоматически:

```
~/Library/Application Support/Orivo/logs/
├── system.log      — системные события и конфигурация
├── jackett.log     — события Jackett
└── torrserver.log  — события TorrServer
```

**Как открыть папку с логами:**
1. Finder → **Перейти → Переход к папке…** (⇧⌘G)
2. Введите: `~/Library/Application Support/Orivo/logs`

Логи автоматически ротируются при достижении 5 МБ.

## Что-то не работает?

Если вы столкнулись с проблемой — создайте [Issue на GitHub](https://github.com/your-org/orivo/issues/new):

1. Опишите проблему: что происходит и при каких условиях
2. Прикрепите скриншот с ошибкой
3. Прикрепите файлы логов из папки `~/Library/Application Support/Orivo/logs/`

Это поможет нам быстро найти и исправить причину.

## Удаление

Удалите `Orivo.app` из папки `/Applications`, а затем папку с данными:

```bash
rm -rf ~/Library/Application\ Support/Orivo
```

## Сборка из исходного кода

Для самостоятельной сборки приложения вам понадобятся установленные **Xcode Command Line Tools** (или Xcode) и компилятор **Swift** (уже встроен в macOS).

1. Клонируйте репозиторий с исходным кодом:
   ```bash
   git clone https://github.com/your-org/orivo.git
   cd orivo
   ```
2. Соберите приложение с помощью скрипта автоматической сборки и упаковки:
   ```bash
   ./build.sh
   ```
3. После завершения сборки готовый бандл `Orivo.app` появится в корневой директории проекта.
